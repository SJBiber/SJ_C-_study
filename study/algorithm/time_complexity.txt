시간 복잡도 
1. 빅 오메가 : 최선일 때의 연산 횟수를 나타낸 표기법    (횟수 : 1번)
2. 빅 세타 : 보통일 때의 연산 횟수를 나타낸 표기법      (횟수 : N/2번)
3. 빅 오 : 최악일 때의 연산 횟수를 나타낸 표기법        (횟수 : N번)

왜 중요한가 ?
 - 같은 기능을 하는 두 알고리즘이라도 하나는 매우 빠르고 , 다른 하나는 느릴 수 있음.
 - 대량의 데이터를 다룰 때 , 효율적인 알고리즘은 실행 속도에 큰 차이를 만듦

대표적인 시간 복잡도 종류

O(1)        상수 시간 - 입력 크기에 상관없이 항상 같은 시간     / 배열에서 특정 인덱스 값 접근
O(log N)    로그 시간 - 입력이 커져도 비교적 천천히 증가        / 이진 탐색
O(N)        선형 시간 - 입력 크기에 비례                       / 배열 전체를 한번 순회
O(N log N)  선형 로그 시간 - 대부분의 정렬 알고리즘은           / 퀵 정렬 , 병합 정렬
O(N^2)      제곱 시간 - 이중 반복문 등                         / 버블 정렬 , 선택 정렬
O(2^N)      지수 시간 - 입력이 조금만 커져도 매우 느려짐        / 재귀적인 파보나치
O(N!)       팩토리얼 시간 - 거의 사용 불가능한 수준             / 순열 생성 알고리즘
